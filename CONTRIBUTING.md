# Contributing to Beaverlib
Contributions to Beaverlib are greatly appreciated! Contributions to Beaverlib can include:
- Year-agnostic contributions
- Main season contributions
- Offseason events contributions

Contributions do not have to be from teams specifically within the Pacific Northwest District; anyone can contribute to this project.

# Rules
The only thing that should ever be contributed to the main branch is year-agnostic changes. These are changes that can be used from year-to-year, not anything season-specific. These changes still must be done through making a branch and then making a Pull Request; no commits can be made directly to main.

For each season, a new branch must be created. This branch will never be merged into main and should solely have features developed for that season only. New branches can also be created for any off-season events. If there are year-agnostic changes, they can be put into a seperate branch and then have a Pull Request made.

In order for a branch to be merged into main, it must have at least one approved review by someone with write access that is not the code owner. If the branch gets a new commit after getting reviewed but before getting merged, it will need to be reviewed again.
